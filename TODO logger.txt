my logger

- le logger a besoin d'un fichier de configuration (d'initialisation) dans lequel on définit pas mal de paramètres : 
	- dossier de logs
	- fichiers de logs
	- si on log sur une console
	- si on log sur un fichier 
- créer des streams dans lesquels on peut log une progression (progress bar)
- configurer le nom de la console
- configurer les erreurs
- gérer les redimensionnements de la console
- gestion de "module" => 1 "module" = 1 tableau. chaque fonctions s'exécutant en parallèle abbonnée va se loguer dans ce tableau
- mettre une limite de lignes de log en fonction de la taille de la console
	- si la console est trop petite, se mettre en mode économie de place
	- si la console a une longue largeur mais petite hauteur -> spliter la console en 2
- le logger gère les erreurs (fichier, ligne, colonne, ...)
- les 'textes' des erreurs sont tous dans un ou plusieurs fichiers 'json' à part. Ces textes sont dynamiques (des variables peuvent intervenir)

- dans le fichier de configuration on met tous les paramètres -> on initialise le logger dans le index.js
